# 커밋 가이드

## 커밋 메시지 형식

```
type: 작업 내용 요약

- 세부 내용 1
- 세부 내용 2
```

type 종류:

- `feat`: 기능 추가
- `fix`: 버그 수정
- `refactor`: 리팩토링
- `chore`: 설정, 의존성 등
- `docs`: 문서
- `test`: 테스트 추가/수정
- `style`: 코드 포맷팅 (기능 변경 없음)

예시:

```
feat: 식당 목록 필터 컴포넌트 구현

- FilterButton 컴포넌트 추가
- 선택 상태에 따른 스타일 분기 처리
- 접근성 속성 추가
```

### 규칙

- 제목은 명령문으로 작성 (과거형 지양)
- 제목은 50자 이내
- 세부 내용은 변경 이유보다 변경 내용 위주로 작성
- 하나의 커밋에 하나의 작업 단위
